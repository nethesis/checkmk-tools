========================================================================================
USE LOCAL SCRIPTS
========================================================================================

NOPOLLING INSTALLATION:
1. Copy script to /usr/lib/check_mk_agent/local/
2. Make executable: chmod +x script_name.sh
3. Run service discovery in CheckMK

EXAMPLE:
cp check_dovecot_status.sh /usr/lib/check_mk_agent/local/
chmod +x /usr/lib/check_mk_agent/local/check_dovecot_status.sh

POLLING INSTALLATION:
1. Copy script to /usr/lib/check_mk_agent/plugins/
2. Make executable: chmod +x script_name.sh
3. Configure execution interval (default: every 5 minutes)

EXAMPLE:
cp check_ransomware_ns7.sh /usr/lib/check_mk_agent/plugins/
chmod +x /usr/lib/check_mk_agent/plugins/check_ransomware_ns7.sh


========================================================================================
USE OF REMOTE SCRIPTS
========================================================================================

INSTALLATION:
1. Copy remote wrapper (rNAME.sh) to /usr/lib/check_mk_agent/local/ or plugins/
2. Make executable: chmod +x rNAME.sh
3. The script will automatically download the latest version from GitHub

NOPOLLING EXAMPLE:
cp rcheck_dovecot_status.sh /usr/lib/check_mk_agent/local/
chmod +x /usr/lib/check_mk_agent/local/rcheck_dovecot_status.sh

POLLING EXAMPLE:
cp rcheck_ransomware_ns7.sh /usr/lib/check_mk_agent/plugins/
chmod +x /usr/lib/check_mk_agent/plugins/rcheck_ransomware_ns7.sh

ADVANTAGES:
 Automatic updates from the repository
 No manual script maintenance
 Centralized management

ADDITIONAL NOTES
========================================================================================

CHECKMK OUTPUT FORMAT:
All scripts follow the CheckMK local check format:
STATUS SERVICE_NAME - DETAILS [| METRICS]

STATES:
0 = OK
1 = WARNING
2 = CRITICAL
3 = UNKNOWN

DIFFERENCE NOPOLLING vs POLLING:
- NOPOLLING: Performed at every check agent (default: every 1 minute)
- POLLING: Performed at configured intervals (typically 5-60 minutes)

BEST PRACTICES:
- Use nopolling for quick checks (service status, counters)
- Use polling for heavy checks (filesystem scan, ransomware)
- Use remote scripts (r*) for multi-server deployments
- Check dependencies before deployment
- Check log file and statefile permissions

========================================================================================
TROUBLESHOOTING
========================================================================================

PROBLEM: Script does not generate output
SOLUTION:
- Check execution permissions: ls -l /usr/lib/check_mk_agent/local/
- Test manually: bash /usr/lib/check_mk_agent/local/script_name.sh
- Check dependencies (systemctl, doveadm, etc.)

PROBLEM: Service does not appear in CheckMK
SOLUTION:
- Run service discovery: Setup → Hosts → Full service scan
- Check agent output: cmk -d hostname
- Check CheckMK log: /omd/sites/SITENAME/var/log/

PROBLEM: Remote Script does not work
SOLUTION:
- Check GitHub connectivity: curl -I https://raw.githubusercontent.com
- Check firewall/proxy
- Test URL manually: curl -fsSL "$SCRIPT_URL"

PROBLEM: Statefile is not created
SOLUTION:
- Check folder permissions: ls -ld /var/lib/check_mk_agent/
- Create manually: mkdir -p /var/lib/check_mk_agent
- Check SELinux: getenforce
