1. check_disk_space.sh
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Rows: 20
Service: (uses standard exit code, not CheckMK local format)
Install Path: /usr/lib/check_mk_agent/plugins/

DESCRIPTION:
Basic script to monitor used disk space on the root (/) filesystem.
Use standard Nagios/CheckMK exit code.

FEATURES:
 Check root disk usage percentage (/)
 Configurable thresholds (default: 80% WARN, 95% CRIT)
 Standard Nagios exit code

CONFIGURATION:
THRESHOLD=80 (warning threshold percentage)

STATUS (EXIT CODE):
0 - OK: Usage < 80%
1 - WARNING: Usage 80-94%
2 - CRITICAL: Usage ≥ 95%

COMMANDS USED:
- df / (root disk space)

OUTPUT EXAMPLE:
"OK - Disk space used: 45%" (exit 0)
"WARNING - Disk space used: 85%" (exit 1)
"CRITICAL - Disk space used: 96%" (exit 2)

NOTES:
 Use standard exit code, not <<<local>>> format
 Monitor root filesystems only /
 Hardcoded thresholds (edit THRESHOLD in script)

Service: SSH_All_Sessions
Install Path: /usr/lib/check_mk_agent/local/

DESCRIPTION:
Counts the total number of active SSH sessions (all users, not just root).

FEATURES:
 Count all active sshd processes (excluding privileged processes)
 Includes sessions from all users
 Generate count metric for charts

STATES:
0 (OK) - No active SSH session
1 (WARN) - One or more active SSH sessions

COMMANDS USED:
- pgrep -af "sshd: "
- grep -v "sshd:.*priv" (exclude privileged/listener processes)

count={number}

"1 SSH_All_Sessions count=3; Active SSH sessions (all users) = 3"

Excludes the master sshd process and privilege separation processes.


4. ssh_root_session
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Lines: 56
Service: SSH_Session
Install Path: /usr/lib/check_mk_agent/local/

DESCRIPTION:
Monitor root SSH events (login, logout, failed attempts) by reading 
/var/log/auth.log and notify each new event.

FEATURES:
 Read log auth.log incrementally (use statefile)
 Detect successful SSH root logins (Accepted)
 Detect SSH root logout (session closed)
 Detect failed attempts (Failed password)
 Extracts source IP
 Store last IP in separate file

FILES USED:
- Log: /var/log/auth.log
- State: /var/lib/check_mk_agent/ssh_root_session.state
- Last IP: /var/lib/check_mk_agent/ssh_root_session.lastip

RESEARCHED PATTERNS:
- "Accepted" + "for root" (login successful)
- "session closed for user root" (logout)
- "Failed password for root"

STATES:
0 (OK) - Logout or no new events
1 (WARN) - Root login successful
2 (CRIT) - Failed root login attempt

STATEFILE LOGIC:
- Stores number of last row processed
- Automatic reset if log rotated (logrotate)
- Process only new lines on each run

OUTPUT EXAMPLE:
"0 SSH_Session - OK: no new root SSH events (last check 2025-10-27 15:30:00)"
"1 SSH_Session - WARN: root SSH login from 192.168.1.50 at Oct 27 14:25:10"
"2 SSH_Session - CRITICAL: Failed root SSH login from 203.0.113.5 at Oct 27 14:30:05"
"0 SSH_Session - OK: root SSH logout (last IP 192.168.1.50) at Oct 27 15:00:00"


5. ssh_fail2ban_status
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Lines: 31
Service: SSH_Fail2ban
Install Path: /usr/lib/check_mk_agent/local/

DESCRIPTION:
Monitor fail2ban jail status for SSH and show number of banned IPs.

FEATURES:
 Check for fail2ban-client command presence
 Query jail "sshd"
 Count currently banned IPs
 Show banned IP list
 Generate banned metrics for charts

CONFIGURATION:
JAIL="sshd"

STATES:
0 (OK) - No IPs banned
2 (CRIT) - One or more IPs banned
3 (UNKNOWN) - fail2ban not found or jail not configured

COMMANDS USED:
- fail2ban-client status sshd

METRICS:
banned={number}

PARSING OUTPUT:
- "Currently banned:" → IP number
- "Banned IP list:" → IP list

OUTPUT EXAMPLE:
"0 SSH_Fail2ban banned=0; OK: no IPs banned by fail2ban (jail=sshd)"
"2 SSH_Fail2ban banned=2; CRITICAL: 2 IPs currently banned by fail2ban (jail=sshd) [203.0.113.5 198.51.100.10]"
"3 SSH_Fail2ban - FAIL: fail2ban-client not found"
"3 SSH_Fail2ban - FAIL: sshd jail not found"

NOTES:
Requires fail2ban installed and configured with sshd jail.

========================================================================================
USE LOCAL SCRIPTS
========================================================================================

POLLING INSTALLATION:

METHOD 1 - CheckMK local format script (ssh_*):
1. Copy script to /usr/lib/check_mk_agent/local/
2. Make executable: chmod +x script_name
3. Run service discovery in CheckMK

EXAMPLE:
cp ssh_sessions_count /usr/lib/check_mk_agent/local/
chmod +x /usr/lib/check_mk_agent/local/ssh_sessions_count

METHOD 2 - Nagios format script (check_disk_space.sh):
1. Copy script to /usr/lib/check_mk_agent/plugins/
2. Make executable: chmod +x script_name.sh
3. Configure in CheckMK as a standard plugin

EXAMPLE:
cp check_disk_space.sh /usr/lib/check_mk_agent/plugins/
chmod +x /usr/lib/check_mk_agent/plugins/check_disk_space.sh


========================================================================================

========================================================================================
TROUBLESHOOTING
========================================================================================

PROBLEM: Script does not generate output
SOLUTION:
- Check execution permissions: ls -l /usr/lib/check_mk_agent/local/
- Test manually: bash /usr/lib/check_mk_agent/local/ssh_sessions_count
- Check shebang: must be #!/usr/bin/env bash or #!/bin/bash

PROBLEM: ssh_root_session is not detecting events
SOLUTION:
- Check path log: ls -l /var/log/auth.log
- On some Ubuntu distros: could be /var/log/auth.log or syslog
- Check log reading permissions
- Test grep: grep "sshd" /var/log/auth.log

PROBLEM: Statefile is not created
SOLUTION:
- Check folder permissions: ls -ld /var/lib/check_mk_agent/
- Create manually: mkdir -p /var/lib/check_mk_agent
- Check owner: chown checkmk:checkmk /var/lib/check_mk_agent

PROBLEM: fail2ban script returns UNKNOWN
SOLUTION:
- Check fail2ban installed: which fail2ban-client
- Check configured jail: fail2ban-client status
- Check jail sshd: fail2ban-client status sshd
- Install if missing: apt install fail2ban

ISSUE: check_disk_space does not appear in CheckMK
SOLUTION:
- Script uses exit code, not local format
- Put it in plugins/ not in local/
- Configure as standard check_mk plugin
- Alternative: convert to local format (edit output)

========================================================================================
SCRIPT CONVERSION FROM PLUGIN TO LOCAL
========================================================================================

EXAMPLE: Convert check_disk_space.sh to local format

BEFORE (Plugin format):
```bash
if [ "$USAGE" -lt "$THRESHOLD" ]; then
    echo "OK - Disk space used: $USAGE%"
    exit 0
elif [ "$USAGE" -lt 95 ]; then
    echo "WARNING - Used disk space: $USAGE%"
    exit 1
else
    echo "CRITICAL - Disk space used: $USAGE%"
    exit 2
fi
```

AFTER (Local format):
```bash
if [ "$USAGE" -lt "$THRESHOLD" ]; then
    echo "0 Disk_Space usage=$USAGE%;80;95; OK - Disk space used: $USAGE%"
elif [ "$USAGE" -lt 95 ]; then
    echo "1 Disk_Space usage=$USAGE%;80;95; WARNING - Disk space used: $USAGE%"
else
    echo "2 Disk_Space usage=$USAGE%;80;95; CRITICAL - Disk space used: $USAGE%"
fi
```

LOCAL FORMAT ADVANTAGES:
 Built-in metrics for charts
 No exit code needed
 Unified output
 More flexible
