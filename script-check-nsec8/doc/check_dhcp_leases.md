# check_dhcp_leases.sh

## Description
Monitor DHCP pool usage on NSecFirewall8, checking the number of active leases against the configured limit.

## Features
- Read the `/tmp/dhcp.leases` file
- Count active (unexpired) leases by comparing timestamps
- Read pool configuration from `uci get dhcp.lan.start/limit`
- Calculate usage percentage
- Threshold: WARNING 80%, CRITICAL 90%

## States
- **OK (0)**: Usage < 80%
- **WARNING (1)**: Usage >= 80% or leases file not found
- **CRITICAL (2)**: Usage >= 90%

## Output CheckMK
```
0 DHCP_Leases active=45;120;135;0;150 Leases active: 45/150 (30%) - OK | active=45 expired=5 total=50 max=150 percent=30
```

## Performance Data
- `active`: Active leases with threshold
- `expired`: Expired leases still in the file
- `total`: Total leases in the file
- `max`: DHCP pool size
- `percent`: Usage percentage

## Requirements
- File `/tmp/dhcp.leases` generated by dnsmasq
- `uci` command to read configuration
- `date` commands to compare timestamps

## Installation
```bash
cp check_dhcp_leases.sh /usr/lib/check_mk_agent/local/rcheck_dhcp_leases.sh
chmod +x /usr/lib/check_mk_agent/local/rcheck_dhcp_leases.sh
```

## Manual testing
```bash
bash /opt/checkmk-tools/script-check-nsec8/full/check_dhcp_leases.sh
```

## dhcp.leases file format
```
1732723456 aa:bb:cc:dd:ee:ff 192.168.1.100 hostname client-id
```
- Field 1: Unix expiration timestamp
- Field 2: MAC address
- Field 3: IP assigned
- Field 4: Hostname
- Field 5: Client ID

## Notes
- Pool full may prevent new device connections
- Static leases do not appear in this file
- Reduce lease time if small pool and many temporary devices
