##  Contesto Ambiente

L'assistente può operare in contesti diversi. Le regole si adattano automaticamente:

| Contesto | Git | Terminale | SSH |
|----------|-----|-----------|-----|
| **VS Code** | GitKraken MCP se disponibile, altrimenti git CLI | Terminale integrato | Via WSL (`wsl -d kali-linux ssh ...`) |
| **Copilot CLI** | git CLI diretto | Bash nativo | SSH diretto (`ssh host ...`) |

-  Eseguire comandi direttamente, evitare risposte solo descrittive
-  Mostrare in chat solo sintesi/esito, non output prolisso
-  Rendere SEMPRE eseguibili gli script: `git update-index --chmod=+x script.sh`

##  Strumenti Preferiti

### Controllo Integrità Repository

- **Script**: `script-ps-tools/check-integrity.ps1`
- **Quando**: dopo modifiche massive, prima di merge, sospetta corruzione

```powershell
.\script-ps-tools\check-integrity.ps1                  # Standard
.\script-ps-tools\check-integrity.ps1 -Detailed        # Lista errori completa
.\script-ps-tools\check-integrity.ps1 -ExportReport    # Esporta su file
.\script-ps-tools\check-integrity.ps1 -Threshold 20    # Soglia custom
.\script-ps-tools\check-integrity.ps1 -SendEmail       # Invia email se errori
```

- Verifica sintassi PowerShell + Bash (`bash -n`)
- Soglia corruzione: 15% (configurabile)
- Exit codes: 0=OK, 1=Warning, 2=Critical
- Integrato con `backup-simple.ps1` (backup bloccato se corruzione >15%)

### Validazione Script

```bash
# Bash/Shell
bash -n script.sh          # Sintassi - exit code DEVE essere 0

# PowerShell
# Usare PSParser::Tokenize() - 0 errori prima di procedere
```

##  Note Specifiche Progetto

### Script CheckMK

- Bash con `set -euo pipefail`
- Logging con funzioni dedicate
- Error handling completo
- Versioning obbligatorio: variabile `VERSION` in ogni script

### Script Ydea

- API authentication via `.env`
- Token management automatico
- Cache in `/tmp/ydea-cache/`

### Backup automatico

- Ogni ora via Windows Scheduled Task
- Script: `backup-sync-complete.ps1`
- Locale: `C:\CheckMK-Backups\`
- Rete: `\\192.168.10.132\usbshare\CheckMK-Backups\`

### Repository locale su host remoti

- Path: `/opt/checkmk-tools/` (git clone su tutti gli host)
- Aggiornamento: `cd /opt/checkmk-tools && git pull`
- READ-ONLY: non modificare file in `/opt/checkmk-tools/`
