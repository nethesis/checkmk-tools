## Environment Context

The assistant can operate in different contexts. The rules adapt automatically:

| Context | Git | Terminal | SSH |
|----------|-----|-----------|-----|
| **VS Code** | GitKraken MCP if available, otherwise git CLI | Integrated terminal | Via WSL (`wsl -d kali-linux ssh ...`) |
| **Copilot CLI** | git CLI direct | Native Bash | Direct SSH (`ssh host ...`) |

- Execute commands directly, avoid only descriptive answers
- Show only summary/result in chat, not verbose output
- ALWAYS make scripts executable: `git update-index --chmod=+x script.sh`

## Favorite Tools

### Repository Integrity Check

- **Script**: `script-ps-tools/check-integrity.ps1`
- **When**: after massive changes, before merge, suspected corruption

```powershell
.\script-ps-tools\check-integrity.ps1 # Standard
.\script-ps-tools\check-integrity.ps1 -Detailed # Complete error list
.\script-ps-tools\check-integrity.ps1 -ExportReport # Export to file
.\script-ps-tools\check-integrity.ps1 -Threshold 20 # Custom threshold
.\script-ps-tools\check-integrity.ps1 -SendEmail # Send email if errors
```

- PowerShell + Bash syntax check (`bash -n`)
- Corruption threshold: 15% (configurable)
- Exit codes: 0=OK, 1=Warning, 2=Critical
- Integrated with `backup-simple.ps1` (backup blocked if corruption >15%)

### Script Validation

```bash
# Bash/Shell
bash -n script.sh # Syntax - exit code MUST be 0

# PowerShell
# Use PSParser::Tokenize() - 0 errors before proceeding
```

## Cleaning Temporary Folders

**Before each commit:**

- Remove: `REPAIR*`, `TEMP*`
- Keep: `BACKUP_ORIGINALI_PROD*`, `*.BACKUP_*`, `*.ORIGINAL`

**File repair workflow:** create `REPAIR_*` → work → show report → wait for approval → replace → remove `REPAIR_*` → commit

## Project Specific Notes

### CheckMK scripts

- Bash with `set -euo pipefail`
- Logging with dedicated functions
- Complete error handling
- Versioning required: `VERSION` variable in each script

### Ydea script

- API authentication via `.env`
- Automatic token management
- Cache in `/tmp/ydea-cache/`

### Automatic backup

- Hourly via Windows Scheduled Task
- Script: `backup-sync-complete.ps1`
- Local: `C:\CheckMK-Backups\`
- Network: `\\192.168.10.132\usbshare\CheckMK-Backups\`

### Local repository on remote hosts

- Path: `/opt/checkmk-tools/` (git clone on all hosts)
- Update: `cd /opt/checkmk-tools && git pull`
- READ-ONLY: Do not edit files in `/opt/checkmk-tools/`
