## Auto-Sync System Active

**CRITICAL:** This repository has an **automatic auto-sync** system configured on remote servers!

### Sync architecture

```
GitHub (Coverup20/checkmk-tools)
    ↓ [auto-git-sync.service]
/opt/checkmk-tools/ (on CheckMK servers)
    ↓ [execute script]
Production
```

### How It Works

1. **Central Repository:** `https://github.com/Coverup20/checkmk-tools`
2. **Server Local Copy:** `/opt/checkmk-tools/` (on each CheckMK server)
3. **Auto-Sync Script:** `script-tools/full/sync_update/auto-git-sync.sh` (run as systemd service)
4. **Sync Frequency:** Every 5-15 minutes (configurable)

### Editing Workflow

```mermaid
1. Edit Local Files (Windows)
   ↓
2. Commit + Push to GitHub
   ↓
3. WAIT auto-sync (5-15 min)
   ↓
4. Updated files in /opt/checkmk-tools/
   ↓
5. Scripts can be executed
```

### Important Implications

- **Never directly edit** files in `/opt/checkmk-tools/` - they will be overwritten!
- **After committing to GitHub**, wait a few minutes before testing on the server
- **For immediate update**, manually run: `sudo bash /opt/checkmk-tools/script-tools/full/sync_update/auto-git-sync.sh`
- **Local scripts may be old** if auto-sync hasn't happened yet

### Remote vs Local Scripts

**Remote Launchers (script-tools/remote/):**
- They ALWAYS download the latest version from GitHub
- Don't depend on auto-sync
- Use for critical/test runs
- Example: `rsetup-auto-upgrade-checkmk.sh`

**Script Full (script-tools/full/<category>/):**
- Run from `/opt/checkmk-tools/`
- Depend on auto-sync to be updated
- Used for local/crontab executions
- Example: `upgrade_maintenance/setup-auto-upgrade-checkmk.sh`

## Best Practices for Copilot

### Before Suggesting Changes

1. **Check** if remote launcher exists for that script
2. **Warn** user to wait for auto-sync after commit
3. **Prefer** to suggest running from GitHub whenever possible
4. **Remember** that local changes on the server will be lost

### When You Make Changes

```bash
# Correct workflow:
1. git add <file>
2. git commit -m "message"
3. git push
4. WAIT for auto-sync (or force manually)
5. Test on the server
```

### Useful Commands

```bash
# Force manual sync
sudo bash /opt/checkmk-tools/script-tools/full/sync_update/auto-git-sync.sh

# Check if file is updated
grep -n "UNIQUE_PATTERN" /opt/checkmk-tools/path/to/script.sh

# See last sync
sudo systemctl status auto-git-sync.service

# See auto-sync log
sudo journalctl -u auto-git-sync.service -n 50
```
