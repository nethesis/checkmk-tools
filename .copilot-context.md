## 🔄 Sistema Auto-Sync Attivo

**⚠️ CRITICO:** Questo repository ha un sistema di **auto-sync automatico** configurato sui server remoti!

### Architettura di Sync

```
GitHub (Coverup20/checkmk-tools)
    ↓ [auto-git-sync.service]
/opt/checkmk-tools/ (sui server CheckMK)
    ↓ [esecuzione script]
Produzione
```

### Come Funziona

1. **Repository Centrale:** `https://github.com/Coverup20/checkmk-tools`
2. **Copia Locale Server:** `/opt/checkmk-tools/` (su ogni server CheckMK)
3. **Script Auto-Sync:** `script-tools/full/auto-git-sync.sh` (eseguito come servizio systemd)
4. **Frequenza Sync:** Ogni 5-15 minuti (configurabile)

### Workflow di Modifica

```mermaid
1. Modifica file locali (Windows)
   ↓
2. Commit + Push a GitHub
   ↓
3. ATTENDI auto-sync (5-15 min)
   ↓
4. File aggiornati in /opt/checkmk-tools/
   ↓
5. Script possono essere eseguiti
```

### ⚠️ Implicazioni Importanti

- **Non modificare mai direttamente** file in `/opt/checkmk-tools/` - verranno sovrascritti!
- **Dopo commit su GitHub**, aspettare qualche minuto prima di testare sul server
- **Per aggiornamento immediato**, eseguire manualmente: `sudo bash /opt/checkmk-tools/script-tools/full/auto-git-sync.sh`
- **Gli script locali possono essere vecchi** se auto-sync non è ancora avvenuto

### Script Remoti vs Locali

**Launcher Remoti (script-tools/remote/):**
- Scaricano SEMPRE l'ultima versione da GitHub
- Non dipendono da auto-sync
- Usare per esecuzioni critiche/test
- Esempio: `rsetup-auto-upgrade-checkmk.sh`

**Script Full (script-tools/full/):**
- Eseguiti da `/opt/checkmk-tools/`
- Dipendono da auto-sync per essere aggiornati
- Usati per esecuzioni locali/crontab
- Esempio: `setup-auto-upgrade-checkmk.sh`

## 🎯 Best Practices per Copilot

### Prima di Suggerire Modifiche

1. ✅ **Verifica** se esiste launcher remoto per quello script
2. ✅ **Avvisa** l'utente di attendere auto-sync dopo commit
3. ✅ **Preferisci** suggerire esecuzione da GitHub quando possibile
4. ✅ **Ricorda** che modifiche locali su server verranno perse

### Quando Committi Modifiche

```bash
# Workflow corretto:
1. git add <file>
2. git commit -m "messaggio"
3. git push
4. ATTENDERE auto-sync (oppure forzare manualmente)
5. Testare sul server
```

### Comandi Utili

```bash
# Forzare sync manuale
sudo bash /opt/checkmk-tools/script-tools/full/auto-git-sync.sh

# Verificare se file è aggiornato
grep -n "PATTERN_UNIVOCO" /opt/checkmk-tools/path/to/script.sh

# Vedere ultimo sync
sudo systemctl status auto-git-sync.service

# Vedere log auto-sync
sudo journalctl -u auto-git-sync.service -n 50
```
